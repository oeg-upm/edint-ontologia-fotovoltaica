Photovoltaic: Description back to ToC

Figure 1 shows the conceptual model of the ontology with its classes and relationships (object properties).

The ontology reuses sensor-related concepts from the Semantic Sensor Network Ontology (Semantic Sensor Network Ontology) to represent the electricity generation of a photovoltaic installation. An infrastructure element is any piece of equipment or urban element that a photovoltaic installation may have. These installations are the features of interest of the measurements of electricity generation made with sensors, which in turn observe an observable property such as the electricity generated, consumed, stored, and fed into the grid.

A photovoltaic meter is a sensor that measures variables related to the electricity generated by the installation.

These generation measurements reflect the amount of electricity generated, consumed, stored, or fed into the grid over a certain period of time. To represent these measurements, the Quantities, Units, Dimensions, and DataTypes Ontology Quantities, Units, Dimensions and DataTypes Ontology is reused. The measurement value is represented by the class QuantityValue which expresses the magnitude and type of a quantity and is obtained by multiplying a numerical value n by a unit of measurement obtained through the property hasUnit which is related to the class Unit that contains a catalog of units of measurement, for example kilowatt-hour (unit:KiloW-HR) used to measure the generation of electricity. The measured period is represented by the class Interval where its start and end times are related to a point in time (Instant). The status of the photovoltaic installation is a taxonomy that includes concepts such as "Operational", "Partial", or "Out of service". The installation has a usage type which can be, for example, "On-grid", "Off-grid", or "Hybrid". The contract has a holder and applies to a specific photovoltaic installation. The contract is also associated with a tariff.
